## 5.range语句（要求都为整形）

range语句可以形成一个数字序列。

序列：有序的集合，比如：`1, 2, 3, 4, 5, ...` 就是数字序列

range语句形成的数字序列是`可迭代类型`（其本质是一个迭代器对象，等同于list列表）

### 语法1：

`range(num)`：形成从`0` 开始，到`num` 结束（不包括num本身）的数字序列

* 如：`range(5)`结果是：0, 1, 2, 3, 4

### 语法2：

`range(num1, num2)` ：形成从num1开始，到num2结束（不含num2）的数字序列

* 如：`range(5, 10)` 结果是：5, 6, 7, 8, 9

### 语法3：

`range(num1, num2, step)` ：形成从num1开始，到num2结束（不含num2）的数字序列，步长以step决定

* 如：`range(5, 10, 2)`结果是：5, 7, 9 (注意步长是2)

## 18.判断回文

切片（Slice）是Python中操作序列的强大工具，基本语法是：

```python
序列[起始位置:结束位置:步长]
```

常见用法示例

```python
s = "Hello World"

# 1. 基本切片 [开始:结束] - 左闭右开
s[0:5]    # "Hello"  - 索引0到4
s[6:11]   # "World"  - 索引6到10
s[:5]     # "Hello"  - 省略开始，默认从0开始
s[6:]     # "World"  - 省略结束，默认到最后

# 2. 负数索引 - 从右往左数
s[-5:]    # "World"  - 最后5个字符
s[:-6]    # "Hello"  - 从开头到倒数第6个（不含）

# 3. 步长 [开始:结束:步长]
s[::2]    # "HloWrd" - 每2个取1个
s[1:8:2]  # "el o"   - 索引1到7，每2个取1个

# 4. 负数步长 - 反向
s[::-1]   # "dlroW olleH" - 字符串反转！
s[10:4:-1] # "dlro"  - 从索引10到5，反向

# 5. 省略所有参数
s[:]      # "Hello World" - 完整复制
s[::]     # "Hello World" - 同上
```

回文判断就用这个：

```python
s = "12321"
s == s[::-1]  # True，因为反转后还是"12321"

s = "hello"
s == s[::-1]  # False，反转后是"olleh"
```

记忆技巧：

* **`[::]`** - 原样

* **`[::-1]`** - 反转

* **`[::2]`** - 跳一个取一个

* **`[1:]`** - 去掉第一个

* **`[:-1]`** - 去掉最后一个

## 27.循环嵌套

* **问题**：在嵌套循环中，如果内外层使用相同的循环变量名，内层会覆盖外层的变量值，导致外层循环逻辑错误。

* **正确做法**：内外层循环应使用不同的变量名，或者内层使用无关紧要的变量名

* **通用原则**：避免变量名冲突，保持每个变量的职责单一、清晰。

  ```python
  # ❌ 错误写法：in 用于两个整数
  elif i-1 in huanfang[0][n-1]:
  # huanfang[0][n-1] 是一个整数（比如3）
  # i-1 也是一个整数（比如2）
  # 不能对整数使用 in
  
  # ✅ 正确写法：直接用 == 比较
  elif i-1 == huanfang[0][n-1]：
  
  huanfang[0] = [8, 1, 6]  # 这是一个列表！
  
  # ✅ 正确：判断整数是否在列表中
  if 1 in huanfang[0]:  # 1 在 [8, 1, 6] 中 → True
  if 9 in huanfang[0]:  # 9 不在 [8, 1, 6] 中 → False

  if i-1 in huanfang[0] and i-1 not in huanfang[0][n-1]:
  ```

- 多个条件的时候不能省略主语
  
  ```python
  # ❌ 错误写法：省略了主语
  if i-1 in huanfang[0] and not in huanfang[0][n-1]:
  
  # ✅ 正确写法：必须写完整
  if i-1 in huanfang[0] and i-1 ！= huanfang[0][n-1]:
  ```

## 1.滑动窗口

滑动窗口详解

滑动窗口是一种处理**连续子数组/子串**问题的高效技巧，能让时间复杂度从 O(n×m) 降到 O(n)。

* * *

一、核心思想

**不要每次都重新计算整个窗口的和，而是利用上一个窗口的结果：**

```python
上一个窗口: [a, b, c]  → 和 = a + b + c
新窗口: [b, c, d] → 和 = 上一个和 - a + d （减左边 + 加右边）

就像窗口在数组上"滑动"，每次只改变两个元素。
```

* * *

二、图解过程（n=8, m=3）

```python
数组:  [3, 1, 4, 1, 5, 9, 2, 6]
索引:   0  1  2  3  4  5  6  7

步骤1: 计算第一个窗口
      [3, 1, 4]  → 和 = 8
       ↑     ↑
       i-m   i

步骤2: i=3，向右滑动一格
      [1, 4, 1]  → 和 = 8 - 3 + 1 = 6
          ↑     ↑
          i-m   i

步骤3: i=4，再滑动
      [4, 1, 5]  → 和 = 6 - 1 + 5 = 10
             ↑     ↑
             i-m   i

步骤4: i=5
      [1, 5, 9]  → 和 = 10 - 4 + 9 = 15

步骤5: i=6
      [5, 9, 2]  → 和 = 15 - 1 + 2 = 16

步骤6: i=7（最后一个窗口）
      [9, 2, 6]  → 和 = 16 - 5 + 6 = 17
```

* * *

### 三、代码模板

```python
n, m = map(int, input().split())
a = [int(input()) for _ in range(n)]

# 1. 计算第一个窗口
window_sum = sum(a[:m])
min_sum = window_sum

# 2. 滑动窗口
for i in range(m, n):
    window_sum = window_sum - a[i - m] + a[i]  # 减旧加新
    min_sum = min(min_sum, window_sum)          # 更新最小值

print(min_sum)
```

### 四、公式对照

| 变量           | 含义      | 第一次循环 (i=3)      |
| ------------ | ------- | ---------------- |
| `a[i-m]`     | 被移出的旧元素 | a[3-3] = a[0]    |
| `a[i]`       | 新加入的元素  | a[3]             |
| `window_sum` | 更新后的窗口和 | 旧和 - a[0] + a[3] |
